About Us

We are a team of undergraduate engineering students from Florida Tech and FSU competing in the national Intelligent Ground Vehicle Competition at Oakland University. Our goal is to build a fully autonomous robot capable of completing the advanced course. Here is a copy of this past competition's rules where you can also find the course descriptions: IGVC Rules 2016

Design Challenges

  • Image Processing
  • Computer Vision
  • Intelligent Navigation
  • Artificial Intelligence
  • Reliability Engineering
  • Vehicle Suspension
  • Parallel Processing
  • Vehicle Design and Assembly

Work Distribution

We have split the major design tasks to complement both teams' strengths. Florida Tech will be handling most of the software including image processing, vision, and navigation. Florida State will build the physical robot including motors, motor control, and weatherproofing. Both teams are responsible for software/hardware integration.

The main subtasks and topics for Florida Tech include: software design, software testing, sensor integration, AI, image processing, and computer vision.

The main subtasks and topics for Florida State include: vehicle design, vehicle assembly, embedded programming, motor control, and vehicle suspension.

A copy of Florida Tech's summer goals is available here. It is written from a software perspective, but reflects the overall goals.

Software and Tools

To construct the robot we are using a variety of software and sensors. Many of these tools are focused on easing our collaboration.

Administratively the teams use Slack, OneNote, and OneDrive to coordinate from remote locations. Additionally the robot is being designed using OnShape. OnShape is a cloud based, open source tool for collaborative CAD design.

A list of major software being used includes:

  • Jenkins as our Continuous Integration Server
  • Gradle for building and testing our code
  • We are considering TestNG, GoogleTest, and nose for use as our testing frameworks
  • RabbitMQ for communication between functional units
  • ROS, OpenCV, and PCL for image processing

The hardware we have chosen (so far) includes:

  • ZED Stereoscopic camera for course detection
  • Nvidia TX1 for image processing, AI, and embedded control
  • Arduino Mega for motor control
